# 1. Frame-Relay 개념 정리

## 🔹 WAN Layer 2 Protocol

- Router ↔ Router의 **Serial Interface**를 연결하여 통신할 때 사용되는 Protocol
- 종류: `HDLC`, `PPP`, `ATM`, `Frame-Relay` 등

---

## 🔹 Frame-Relay 개념 및 명칭 정리

- Router ↔ Router 사이의 연결이 Connected될 때 사용되는 Protocol
- Router ↔ Router 사이에서 **Frame-Relay Switch**를 사용하여 회선 Switching을 통해 데이터를 전달하는 Protocol
- Frame-Relay는 **DLCI(Data-Link Connection Identifier)** 라는 식별 Code를 사용한다
- DLCI는 회선을 식별하는 번호이며, 각 **VC(Virtual-Circuit)** 마다 다른 DLCI가 부여된다

### ✅ 장점
- 전용 회선을 설치한 것과 같은 서비스를 받을 수 있으면서, 공용 가상 회선을 통해서 사용이 가능하다
- Layer 2 수준 서비스를 실시하기 때문에 다음대 비해 속도가 빠르다
- 하나의 인터페이스 및 회선을 이용하여 **Multi-access**가 가능하다  
  [EX: 공공도서관(공용)에 설치되어 있는 회선]

### ⚠ 단점
- 다수의 사용자가 공동으로 사용하기 때문에 보안에서 취약하며,  
  동시에 사용자 사용하게되면 신뢰성과 대역폭의 사용효율이 낮다

---

## 🔹 Frame-Relay Virtual Circuit을 사용하여 통신을 실시

- Frame-Relay의 가상의 회선을 **VC(Virtual Circuit)** 라고 한다
- VC는 영구적인 연결을 실시한 **PVC**와 자동으로 통신을 실시하는 **SVC**의 형식이 존재한다

| 종류 | 설명 |
|------|------|
| **PVC** (Permanent VC) | 영구적 가상회선으로 통신량이 일어나는 곳일수록 효율적인 방식 |
| **SVC** (Switched VC) | 교환식 가상회선으로 일시적인 통신이 실시되는 곳일수록 효율적인 방식 |

---

## 🔹 Frame-Relay LMI (Local Management Interface)

- Router와 Frame-Relay Switch 사이에서 **Virtual-Circuit에 관한 정보를 교환**할 때 사용되는 Protocol
- Router는 Frame-Relay Switch의 Virtual-Circuit 상태를 확인할 수 없기 때문에 직접 연결된 Frame-Relay Switch와 Virtual-Circuit 상태를 주기적으로 확인할 때 사용된다
- 기본적인 동작은 정보의 규격화된 전달로써 DLCI에 대한 상태를 전달하며 시그널 메세지를 사용하여 동작 유/무를 주기적으로 확인한다
- LMI는 CISCO Router의 기본값 **AUTO**로 설정되어 있으며 Frame-Relay Switch의 LMI Type으로 결정된다

### LMI Type
- `Cisco`
- `ANSI`
- `Q933A`

```cisco
interface serial 1/x
 frame-relay lmi-type [cisco | ansi | q933a]
```

## 🔹 Frame-Relay Status

| 상태 | 설명 |
|------|------|
| **Active** | Frame-Relay VC가 사용 중 (통신이 가능한 상태) |
| **Inactive** | Local Router와 FR-SW사이는 실시되었으나 FR-SW와 Remote Router의 연결이 안되었을 경우 |
| **Deleted** | Frame-Relay통신이 실시되지 않은상태 (L2 Protocol이 다르거나 LMI Type이 다른경우) |
